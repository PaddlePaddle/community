## 🗓 项目节奏

每个周期：一个人 独立在 **4周完整1个产品**
* 报名&面试：提交简历 + 想法 -> 参与30 分钟面试 -> 主要考察对造出一个优秀产品和提高自己影响力是否有热情
* Week 1：选题 +  调研 + 方案设计 输出 PRD 文档(含 MVP)
* Week 2：用 Vibe Coding 工具专注 MVP 开发 + 测试(功能测试/集成测试)
* Week 3：发布 Pre 版本，邀请测内用户试用拿到反馈，进一步优化
* Week 4：发布 Release 版本， 撰写 PR 文章宣发，做用户增长/收集用户反馈，闭环产品用户增长飞轮

你的节奏我们会保证可控，不卷，只专注于提升真实能力。

## 项目评判
### 核心评判原则
1. **真实价值优先**：拒绝炫技式Demo/玩具项目，核心评判产品是否解决真实场景痛点，具备可落地的用户使用价值；
2. **全周期闭环**：覆盖「选题调研→方案设计→开发测试→用户验证→发布宣发→迭代运营」完整产品闭环，而非仅完成代码开发；
3. **开源生态导向**：符合开源合规要求，具备社区可维护性、可扩展性，鼓励社区共建，而非一次性交付的闭源式项目；
4. **用户增长导向**：以真实用户行为、有效反馈为核心评判依据，而非自嗨式产品交付。

---
### 一票否决项（满足任意一项，直接判定为不通过）
1. 项目未完整开源至GitHub等合规开源平台，或代码存在抄袭、侵权
2. 未使用PaddleOCR或ERNIE相关核心开源组件之一
3. 产品核心功能完全不可用，存在阻断性bug导致核心流程无法跑通，无实际使用价值；
4. 未完成4周周期核心交付节点，无完整PRD文档、无测试记录、无正式Release发布版本；
5. 宣发内容、用户反馈、社区数据存在造假、刷量等作弊行为。

---
### 等级判定规则
1. **过关等级**：同时满足
   - 无任何一票否决项；
   - 下文5个评判维度的**所有过关标准100%全部满足**。
2. **优秀等级**：同时满足
   - 无任何一票否决项；
   - 下文5个评判维度的**所有过关标准100%全部满足**；
   - 下文5个评判维度的**优秀标准满足率≥80%**，且必须100%满足「社区生命力与增长飞轮」维度的全部优秀标准（硬性门槛）。
