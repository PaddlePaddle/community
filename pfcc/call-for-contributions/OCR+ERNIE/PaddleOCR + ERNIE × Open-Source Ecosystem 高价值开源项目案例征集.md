### 开发节奏

在确认入选后，每个人将独立在 4～8 周内完整交付一个产品。参考节奏如下：

* **Week 1～2：产品定义期**
    * 完成选题、市场调研与方案设计。
    * 输出规范的 PRD 文档（含核心 MVP 边界定义）。
* **Week 3～4：敏捷开发期**
    * 使用 Vibe Coding 工具专注 MVP 开发。
    * 完成核心功能测试与集成测试。
* **Week 5～6：内测迭代期**
    * 发布 Pre 版本。
    * 邀请内测用户试用，拿到真实反馈并进一步优化。
* **Week 7～8：发布与增长期**
    * 发布 Release 正式版本。
    * 撰写 PR 技术文章宣发，进行用户增长，收集反馈，闭环产品飞轮。

## ⚖️ 评审规则与标准

### 🚫 一票否决项（触发任意一项，直接不通过）
* 项目未完整开源至 GitHub 等合规平台，或代码存在抄袭、侵权。
* 未使用 PaddleOCR 或 ERNIE 相关的核心开源组件。
* 产品核心功能完全不可用，存在阻断性 Bug，无实际使用价值。
* 未完成开发周期的核心交付节点（无完整 PRD、无测试记录、无正式 Release 版本）。
* 宣发内容、用户反馈、社区数据存在造假、刷量等作弊行为。

### 🏆 等级判定规则
* **过关等级**：无一票否决项 + 下方评判维度表中【过关标准】100% 满足。
* **优秀等级**：达到过关等级 + 下方评判维度表中的【优秀标准】满足率 ≥80%，**且必须 100% 满足**「维度 5：社区生命力与增长飞轮」的全部优秀标准（此为硬性门槛）。
